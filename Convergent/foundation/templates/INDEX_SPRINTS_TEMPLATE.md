# INDEX_SPRINTS
#### Description: Template for the sprints index file. INDEX_SPRINTS is the master registry tracking all sprints by ID, status, and name. Sprint statuses follow a separate lifecycle: Planned → In Progress → Done. Update points are defined in the Sprint Playbook.
#### Convergent: v1.3.2
#### governs: <project_name>

> **FOUNDATION RULE**
> INDEX integrity is mandatory.
> Status must match between internal file metadata and INDEX row.
> SP defines update points per phase.

## How to use
1. Create sprint from `foundation/templates/SPRINT_DESCRIPTION_TEMPLATE.md`
2. Link sprint documentation from planning phase
3. Update status below
