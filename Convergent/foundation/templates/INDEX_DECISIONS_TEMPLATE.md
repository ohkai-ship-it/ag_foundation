# INDEX_DECISIONS
#### Description: Template for the decisions index file. INDEX_DECISIONS is the master registry tracking all ADRs by status, priority, and ownership. INDEX row status must always match the internal Status field in the corresponding ADR file. Update points are defined in the Sprint Playbook.
#### Convergent: v1.3.2
#### governs: <project_name>

> **FOUNDATION RULE**
> INDEX integrity is mandatory.
> Status must match between internal file metadata and INDEX row.
> SP defines update points per phase.

---

## How to use
1. Create ADR from `foundation/templates/ADR_TEMPLATE.md`
2. Link ADR from the relevant AF item + PR
3. Update status below
