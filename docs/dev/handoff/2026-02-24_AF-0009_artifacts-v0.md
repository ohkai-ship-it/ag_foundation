## Summary
Implemented artifact registry v0 with artifact indexing, automatic `result.md` generation on run completion, and `ag artifacts list` CLI command with human and JSON output support.

## What Was Done

### 1. Artifact Registration in Runtime (`src/ag/core/runtime.py`)
- V0Orchestrator now creates `result.md` artifact after each run
- Artifact contains step summaries in Markdown format
- Artifact ID format: `{run_id}-result`
- Type: `text/markdown`

### 2. `_build_result_artifact()` Method
Generates Markdown content with:
- Run metadata (status, mode, duration, playbook)
- Step-by-step summaries with status indicators
- Error details for failed steps

### 3. `ag artifacts list` Command (`src/ag/cli/main.py`)
- Required options: `--run <run_id>`, `--workspace <workspace_id>`
- Human output: Rich table with ID, type, size, path
- JSON output: Array of artifact objects
- Empty list for non-existent runs (stable behavior)

### 4. Test Suite (`tests/test_artifacts.py`)
11 tests covering:
- Artifact schema validation
- Artifacts table exists in SQLite
- Run creates result artifact automatically
- Result artifact contains step summaries
- CLI command behavior (requires workspace, empty list)
- Integration: run → list artifacts flow
