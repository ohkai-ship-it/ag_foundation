# Handoff Note — AF-0005 — Contracts: TaskSpec + RunTrace + Playbook schemas v0.1
**Date:** 2026-02-24
**Author:** Jacob (Junior Engineer)
**Status:** Ready for review

---

## Summary
Implemented v0.1 schemas for TaskSpec, RunTrace, and Playbook with Pydantic models, builders for fluent construction, and comprehensive contract tests ensuring JSON round-trip stability and additive evolution guardrails.

## What Was Done

### 1. TaskSpec v0.1 (`src/ag/core/task_spec.py`)
- `TaskSpec` model with required fields: `prompt`, `workspace_id`
- `task_spec_version` field (default "0.1")
- `ExecutionMode` enum: `manual`, `supervised`, `autonomous`
- `Budgets` model: `max_steps`, `max_tokens`, `max_duration_seconds`
- `Constraints` model: `allowed_skills`, `blocked_skills`, `allowed_paths`, `blocked_paths`
- `TaskSpecBuilder` for fluent construction
- JSON serialization: `to_json()`, `from_json()`

### 2. RunTrace v0.1 (`src/ag/core/run_trace.py`)
- `RunTrace` model with required fields: `run_id`, `workspace_id`, `started_at`
- `trace_version` field (default "0.1")
- `mode`, `playbook` (PlaybookMetadata), `steps[]`, `artifacts[]`
- `verifier` (VerifierModel with status, checked_at, message, evidence)
- `final` status and `error` field
- Supporting models:
  - `Step`: step_id, step_number, step_type, skill_name, input/output summaries, timestamps, error
  - `Artifact`: artifact_id, path, artifact_type, size_bytes, checksum, metadata
  - `PlaybookMetadata`: name, version
  - `Verifier`/`VerifierModel`: status enum, timestamps, evidence
  - `FinalStatus` enum: `success`, `failure`, `timeout`, `user_cancelled`, `pending`
- `RunTraceBuilder` for fluent construction
- JSON serialization with datetime handling

### 3. Playbook v0.1 (`src/ag/core/playbook.py`)
- `Playbook` model: `name`, `version`, `reasoning_modes`, `budgets`, `steps[]`
- `PlaybookStep` model: name, step_type, skill_name, reasoning_mode, required flag
- `PlaybookStepType` enum: `reasoning`, `action`, `verification`, `synthesis`
- `ReasoningMode` enum: `fast`, `balanced`, `deep`
- Step validation (skill_name required for action steps)
- `PlaybookBuilder` for fluent construction
- JSON serialization

### 4. Contract Tests (`tests/test_contracts.py`)
21 tests covering:
- **TaskSpec:** version field, required fields, JSON round-trip, stable defaults, builder
- **RunTrace:** version field, required fields, JSON round-trip, stable defaults, builder, step artifacts
- **Playbook:** version field, required fields, JSON round-trip, stable defaults, builder, step validation
- **Additive evolution:** guardrails for v0 stability

### 5. Package Exports (`src/ag/core/__init__.py`)
All models and builders exported for easy import.
