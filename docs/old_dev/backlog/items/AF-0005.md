## Problem
Truthful UX and evidence rules require explicit contracts. Prior wording didn’t lock down required fields (e.g., playbook preference) or the contract-test strategy (evolution, serialization, stability).

## Goal
Define v0.1 schemas for TaskSpec, RunTrace, and a minimal Playbook; add builders and contract tests so runtime/CLI are unambiguous and forward-compatible.

## Non-goals
- Full workflow engine schema.
- Retrieval/RAG evidence bundle.
- Telemetry export.

## Acceptance criteria (Definition of Done)
- [ ] TaskSpec v0.1 includes at minimum: `task_spec_version`, `prompt`, `workspace_id`, `mode`, `playbook_preference`, `budgets`, `constraints`.
- [ ] RunTrace v0.1 includes at minimum: `trace_version`, `run_id`, timestamps, `workspace_id`, `mode`, playbook metadata, `steps[]`, `artifacts[]`, `verifier`, `final`.
- [ ] Playbook v0.1 schema exists (minimal): `name`, `version`, `reasoning_modes`, `budgets`, `steps[]` (linear sequence).
- [ ] Contract tests cover: JSON round-trip, version fields present, additive evolution guardrails (no removals/renames within v0), and stable defaults.
- [ ] Schema versioning strategy is documented (or ADR referenced) and enforced by tests.

## Implementation notes
- Use Pydantic (or equivalent).
- Additive-only policy for v0.
- Align fields with CLI truthful labels (mode, verifier.status, timings).

## Risks
P1: schema churn. Mitigate with explicit v0.1 scope + ADR-0002 + contract tests.
